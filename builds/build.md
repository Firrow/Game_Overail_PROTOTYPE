# Gestion des builds
Les builds du jeu sont déposés dans un dossier en fonction de l'objectif du build :
- **unstable** : Tous les builds de tests, version incomplète qui peuvent planter ou disfonctionner
- **pre-release** : Build stabilisé d'une version mineure, déployable à petite échelle pour les bêta-tests
- **release** : Build officiel du jeu, prêt à être déployés et utilisés
